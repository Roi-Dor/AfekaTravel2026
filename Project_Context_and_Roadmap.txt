PROJECT OVERVIEW: Afka Travel Itinerary 2026
[cite_start]This is a Full-Stack Web Development project involving two distinct servers (Express & Next.js) and AI integration[cite: 1, 3].

CORE ARCHITECTURE REQUIREMENTS:
1. Express Server (Auth & Security):
   - [cite_start]Handle user registration and login[cite: 5].
   - [cite_start]Store user data with salted encryption[cite: 5].
   - [cite_start]Issue JWT tokens containing the developers' names[cite: 6].
   - [cite_start]Implement "Silent Refresh" for the JWT once per day[cite: 6].

2. Next.js Server (Frontend & Logic):
   - [cite_start]Use Middleware to protect all routes via the JWT (soft/seamless validation)[cite: 8].
   - [cite_start]Main Title: "מסלול טיולים אפקה 2026"[cite: 9].
   - [cite_start]Pages: Home (index), Planning Page, and History Page[cite: 9, 10, 11].

FUNCTIONAL REQUIREMENTS:
- [cite_start]Planning Page: User inputs Country/City, Trip Type (Trek/Bike), and Duration[cite: 12, 13].
- [cite_start]LLM Integration: All itinerary details must be generated by an AI model[cite: 10].
- [cite_start]Mapping: Use Leaflet.js to display realistic paths (not straight lines)[cite: 10, 18].
- Logistics: 
    * [cite_start]Biking: 30-70 km/day, point-to-point[cite: 16].
    * [cite_start]Trekking: 5-10 km/day, circular routes[cite: 17].
- [cite_start]Visuals: 3-day weather forecast and one characteristic image (AI or Real) per route[cite: 19, 20].
- [cite_start]Database: Save approved itineraries for later retrieval[cite: 21].
